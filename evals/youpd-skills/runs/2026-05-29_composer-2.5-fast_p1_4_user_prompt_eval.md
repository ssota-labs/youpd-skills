## Why user-prompt eval?

Skill evals should mirror production: the subagent receives a **natural user utterance**, loads `SKILL.md` via trigger, and discovers routes — not a checklist like "read analyze-title.md step 3".

**Bad (harness-heavy):** "Run list-analysis-candidates then save-title-analysis per D3 §9-1."

**Good (user-like):** "AI 업무 자동화 레퍼런스 폴더에 넣은 영상들 제목 후크 분포 보여줘."

Non-user setup allowed only for: repo path, temp `--db`, `source .env.local`.
